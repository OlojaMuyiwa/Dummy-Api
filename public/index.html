How it works

DummyAPI can be used with any type of front end project that needs any dummy data in JSON format.
You can use examples below to check how DummyAPI works. Feel free to enjoy it in your awesome projects!

Test Route Example

fetch('/api/v1/app', { method: 'GET', /* or POST/PUT/PATCH/DELETE */ })

.then(res => res.json()) .then(data => console.log(data));

/* { status: 'ok', method: 'GET', // or POST/PUT/PATCH/DELETE } */

All the resources can be used with query params to achieve pagination and get limited data. limit=10 clears the limit and you get all items

But you can specify limit in the query params,limt=50 sets the limit to 50 data, th maximum per request is 100

Limit Example

fetch('/api/v1/app?limit=50', { method: 'GET', /* or POST/PUT/PATCH/DELETE */ })

.then(res => res.json()) .then(data => console.log(data));

/* { status: 'ok', method: 'GET', // or POST/PUT/PATCH/DELETE } */

You can fetch all users using this route `/api/v1/app`, which return a json data including robots placeholder images as well as dummy name, age, email, username, height e.t.c

However you only get data with a limit of 10 unless you specify the limit

Get All Users Example

fetch('/api/v1/app', { method: 'GET', /* or POST/PUT/PATCH/DELETE */ })

.then(res => res.json()) .then(data => console.log(data));

/* { status: 'ok', method: 'GET', // or POST/PUT/PATCH/DELETE } */

You can fetch single user using this route `/api/v1/app/{:id}`, which return a single json data including robots placeholder images as well as dummy name, age, email, username, height e.t.c

Where :id is the id of the user

Get Single user Example

fetch('/api/v1/app/:{id}', { method: 'GET', /* or POST/PUT/PATCH/DELETE */ })

.then(res => res.json()) .then(data => console.log(data));

/* { status: 'ok', method: 'GET', // or POST/PUT/PATCH/DELETE } */

You can fetch single user using this route `/api/v1/app/?{query-params}`, which return a single json data including robots placeholder images as well as dummy name, age, email, username, height e.t.c

Where {query-params} is the query object

Querying Example

fetch('/api/v1/app/?gender=female,firstName=Terry,lastName=Medhurst', { method: 'GET', /* or POST/PUT/PATCH/DELETE */ })
